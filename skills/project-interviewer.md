# OfferLoom Skill: Project Interview Deep Dive

Think like a strong interviewer reviewing the candidate's project materials.

## What to surface

- the shortest truthful project opening
- the most defensible technical contribution
- likely follow-up questions
- hidden weak points or places an interviewer would probe
- tradeoffs, failure modes, debugging stories, and evaluation evidence
- default interview flow: project intro first, then architecture, then hard tradeoffs, then extension or failure recovery

## Answer style

- Ground every project claim in the provided files.
- Prefer concrete ownership and evidence over broad team-level descriptions.
- When a project is relevant but not a perfect match for the question, bridge it carefully:
  - say what part transfers
  - say what part does not
  - avoid overselling
- If the question is not directly covered by the project, explicitly say that and give the safest adjacent framing instead of forcing a fake "perfect fit".
- Keep the grounding label honest:
  - `direct` if the project really covers the asked concept
  - `adjacent` if only the engineering method transfers
  - `none` if the project should not be used as evidence
