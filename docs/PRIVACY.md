# Privacy and Data Handling

OfferLoom 默认就是“公开底座 + 私有工作集”的混合模式，所以边界要清楚。

## 公开数据

这些内容适合随仓库公开发布：

- `sources/documents/*`
- `sources/question-banks/*`
- 公共 GitHub 来源整理出的文档与题库
- 配置说明与脚本

## 私有数据

这些内容默认不应进入仓库：

- `mywork/` 中的真实项目
- 本地 override / runtime 配置
- 基于真实项目生成的答案
- 本地模型缓存
- 私有目录绝对路径

## UI 展示与本地路径

系统会尽量把展示层做成可公开引用：

- UI 中出现的是来源标签、相对路径和逻辑引用
- 本机绝对路径主要留在服务端内部使用

绝对路径只用于：

- 文件监听刷新
- 本地读取文档
- 本机 Codex 执行

## 发布建议

- 不要发布由真实 `mywork` 建出来的数据库
- 不要发布由真实 `mywork` 生成的答案
- 发布前用公开默认源重新建一次库
- 把 `mywork/` 清到只剩 README 占位文件
