# Orbit RAG

## 项目目标

Orbit RAG 是一个面向问答和面试准备的检索增强系统，核心目标是提升检索召回、引用可解释性和答案一致性，而不是单纯追求更长的上下文窗口。

## 你的角色

我主导数据清洗、切块策略、召回排序、评测集设计和失败案例分析，并把结果整理成对外可解释的答题模板。

## 技术方案

整体链路包括文档标准化、章节切分、FTS 词法检索、embedding 召回、启发式重排和答案生成。检索阶段优先召回能直接支撑问题意图的章节，再把项目材料作为 grounding 证据补进来，避免答案只会背基础知识却讲不出真实经历。

## 成果/状态

在样例数据上，系统已经能够把题库题目稳定关联到指南章节，并给出 2 到 4 个项目切入点。相比只做关键词匹配的版本，结果更集中，人工检查成本更低。

## 难点与修复

最大的 tradeoff 在于“召回多样性”和“答案可控性”之间的平衡。召回太宽会让模型引用漂移，召回太窄又容易漏掉关键背景。最终我把 heading 匹配、词法覆盖、语义相似和项目 grounding 组合起来，并为 broad question 额外保留 overview 文档。

## 结果证据

我会在面试里把效果证据分成三类去讲：链接命中质量、引用闭环是否完整、项目叙事是否能自然过渡到通用知识问题。
