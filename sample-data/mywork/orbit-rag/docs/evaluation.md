# Evaluation Notes

## 指标设计

- 检索命中质量：题目是否能命中正确的指南章节。
- 项目 grounding：答案是否能回到真实项目文档，而不是只停在概念解释。
- 可追溯性：回答里的引用能否回到具体章节或项目材料。

## 失败案例

- 纯 broad question 容易把多个项目片段混在一起，回答看起来全面但缺少主线。
- 如果没有 overview 文档，模型会高频使用零散片段，导致开场不自然。
- 当工作材料只有代码没有 README 时，面试回答会缺少“为什么做”和“结果如何”的闭环。

## 结论

对于面试准备场景，好的答案不只是“知识点对”，而是“知识点对 + 引用对 + 项目故事也对”。
