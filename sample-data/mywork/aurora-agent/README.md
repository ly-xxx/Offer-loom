# Aurora Agent

## 项目目标

Aurora Agent 是一个面向知识工作者的多 Agent 工作台，目标是把资料整理、任务规划、工具调用和结果复盘放进同一条工作流里，减少“会说不会落地”的断层。

## 你的角色

我负责主链路设计和端到端落地，重点覆盖任务拆分、上下文打包、引用回链、前端交互和日志观测这几块。

## 技术方案

系统架构分成四层：任务编排层、检索与引用层、执行代理层、结果展示层。编排层把用户意图拆成 planner / researcher / composer 三种角色；检索层负责把指南和项目文档切块并做混合检索；执行层统一接 Codex CLI；展示层用在线文档和可拖拽终端把引用、答案、追问和实时文件刷新放到一个界面里。

## 成果/状态

样例版本里已经可以稳定展示题库到知识锚点的高亮链路，项目问答会自动拼接成可直接演练的完整答案。内部验收时，任务链路命中率和引用完整性都明显高于纯手工整理方式。

## 挑战

最难的问题不是生成答案，而是让答案“可追溯”。如果只给模型扔长上下文，输出很容易泛化且难以检查，所以我把公开引用路径和本机真实路径分开：前者用于页面展示和答案引用，后者只给文件监听和本机执行使用。

## 疑难解答

早期版本里，浏览器开着系统代理时访问本地站点会看到 502。后面我补了启动时的代理提示，并明确区分 127.0.0.1、本机局域网地址和 NO_PROXY 设置，避免误判成服务没启动。
