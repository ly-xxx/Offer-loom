## 1. 大模型推理服务架构

### Q: 设计一个大模型推理服务

```
┌───────────────────────────────────────────────────────────────┐
│                      Load Balancer                             │
└───────────────┬───────────────────────────────┬───────────────┘
                │                               │
┌───────────────▼───────────────┐ ┌─────────────▼───────────────┐
│        API Gateway            │ │        API Gateway           │
│  - 认证鉴权                    │ │  - 限流                      │
│  - 请求路由                    │ │  - 请求队列                   │
└───────────────┬───────────────┘ └─────────────┬───────────────┘
                │                               │
┌───────────────▼───────────────────────────────▼───────────────┐
│                    Request Scheduler                           │
│  - 优先级队列                                                   │
│  - Continuous Batching                                         │
│  - 请求调度                                                     │
└───────────────┬───────────────────────────────┬───────────────┘
                │                               │
┌───────────────▼───────────────┐ ┌─────────────▼───────────────┐
│   Inference Engine (vLLM)     │ │   Inference Engine (vLLM)    │
│   - PagedAttention            │ │   - Tensor Parallelism       │
│   - KV Cache Management       │ │   - KV Cache Management      │
│   - GPU Cluster (4xA100)      │ │   - GPU Cluster (4xA100)     │
└───────────────────────────────┘ └─────────────────────────────┘
```

### 关键设计点

1. **高并发**：请求队列 + Continuous Batching
2. **低延迟**：KV Cache + 投机解码 + 量化
3. **高可用**：多实例 + 自动故障转移
4. **可伸缩**：按流量自动扩缩容

## 3. 分布式训练系统

### Q: 大模型分布式训练的并行策略

| 并行方式 | 原理 | 适用场景 |
|----------|------|----------|
| 数据并行 (DP) | 复制模型到多卡 | 模型放得下单卡 |
| 张量并行 (TP) | 单层切到多卡 | 单层太大 |
| 流水线并行 (PP) | 不同层在不同卡 | 层数多 |
| ZeRO | 切分优化器状态/梯度/参数 | 减少冗余 |
| 序列并行 (SP) | 序列维度切分 | 长序列 |

### Q: DeepSpeed ZeRO 三个阶段

| 阶段 | 切分内容 | 显存节省 |
|------|----------|----------|
| ZeRO-1 | 优化器状态 | 4x |
| ZeRO-2 | + 梯度 | 8x |
| ZeRO-3 | + 模型参数 | N_gpu x |

## 4. Agent 系统设计

### Q: 设计一个可靠的 Agent 系统

**关键组件**：
1. **意图路由**：判断是否需要工具、需要哪些工具
2. **执行引擎**：管理工具调用的生命周期
3. **错误恢复**：重试、回退、人工接管
4. **安全沙箱**：隔离工具执行环境
5. **观测系统**：日志、追踪、告警

```
用户请求 → 意图路由器 → [直接回答 / 工具调用 / 多步推理]
                              ↓
                        执行引擎
                        ├─ 工具注册表
                        ├─ 执行沙箱
                        ├─ 重试机制 (指数退避)
                        └─ 超时控制
                              ↓
                        结果验证 → 输出
```
