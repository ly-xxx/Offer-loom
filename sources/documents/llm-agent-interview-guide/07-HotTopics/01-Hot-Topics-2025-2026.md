## 2. MoE (Mixture of Experts)

### Q: MoE 的原理 ⭐⭐⭐

**核心思想**：每个 token 只激活部分专家，实现"大模型参数量、小模型计算量"。

```
输入 → Router → 选择 Top-K 专家
              → Expert 1 (选中) ──┐
              → Expert 2          │ → 加权求和 → 输出
              → Expert 3 (选中) ──┘
              → Expert 4
```

**Router 公式**：

$$G(x) = \text{TopK}(\text{softmax}(x \cdot W_g))$$

**输出**：

$$y = \sum_{i \in \text{TopK}} G(x)_i \cdot E_i(x)$$

### Q: MoE 训练的挑战

1. **负载均衡**：避免所有 token 都路由到同一专家
   - Auxiliary Loss：惩罚不均衡的路由

$$\mathcal{L}_{aux} = \alpha \cdot N \sum_{i=1}^{N} f_i \cdot P_i$$

2. **通信开销**：Expert Parallelism 需要 All-to-All 通信

3. **训练不稳定**：路由决策的离散性

## 3. 推理模型 (Reasoning Models)

### Q: o1/o3 vs DeepSeek-R1 的区别

| 特性 | OpenAI o1/o3 | DeepSeek-R1 |
|------|-------------|-------------|
| 训练方法 | 未公开 | GRPO + RL |
| 开源 | ❌ | ✅ |
| 推理方式 | 隐式思考 | 显式 CoT |
| 思考过程 | 隐藏 | 可见 |
| 成本 | 高 | 低 |

### Q: Test-Time Compute 是什么？

在推理时投入更多计算资源来提升性能：
- **Best-of-N**：生成多个回答选最好的
- **长思考**：更长的 CoT
- **自我验证**：生成后自我检查
- **搜索**：Tree-of-Thought 等搜索策略
