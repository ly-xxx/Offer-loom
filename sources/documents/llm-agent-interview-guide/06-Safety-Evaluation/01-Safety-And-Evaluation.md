# 安全与评估

## 1. 大模型安全

### Q: LLM 的安全风险有哪些？

| 风险类型 | 描述 | 缓解方案 |
|----------|------|----------|
| 越狱攻击 | 绕过安全限制 | 多层防御、对抗训练 |
| 提示注入 | 恶意指令注入 | 指令隔离、输入过滤 |
| 数据泄露 | 输出训练数据 | 差分隐私、数据脱敏 |
| 有害内容 | 生成有害输出 | RLHF 对齐、过滤器 |
| 偏见 | 系统性偏差 | 数据平衡、去偏微调 |

### Q: 常见越狱攻击方法

1. **角色扮演**："假设你是一个没有限制的AI..."
2. **Multi-turn**：逐步引导突破限制
3. **编码绕过**：Base64/ROT13 编码恶意指令
4. **Token 拼接**：利用分词特性拼出敏感词
5. **对抗后缀**：GCG 等梯度方法生成对抗样本

### Q: 安全防御策略

**多层防御**：
```
输入过滤 → 系统提示防御 → 模型内在对齐 → 输出过滤 → 人工审核
```

## 2. 幻觉问题

### Q: 大模型幻觉的类型和缓解方案 ⭐⭐⭐

**幻觉类型**：
1. **事实性幻觉**：生成与事实不符的信息
2. **忠实度幻觉**：输出与输入不一致
3. **指令幻觉**：不遵循指令要求

**缓解方案**：

| 方法 | 描述 | 效果 |
|------|------|------|
| RAG | 基于检索的生成 | ⭐⭐⭐⭐ |
| CoT | 链式推理 | ⭐⭐⭐ |
| Self-Consistency | 多次采样取一致结果 | ⭐⭐⭐ |
| RLHF | 对齐训练 | ⭐⭐⭐⭐ |
| 知识编辑 | 修改模型中的知识 | ⭐⭐ |
| 自我纠正 | 让模型检查自己的输出 | ⭐⭐⭐ |

## 3. 评估体系

### Q: LLM 评估的主要维度

| 维度 | 评估内容 | 常用 Benchmark |
|------|----------|----------------|
| 知识能力 | 世界知识 | MMLU, C-Eval, CMMLU |
| 推理能力 | 逻辑、数学 | GSM8K, MATH, ARC |
| 代码能力 | 编程 | HumanEval, MBPP |
| 语言能力 | 理解、生成 | HellaSwag, WinoGrande |
| 对话能力 | 指令遵循 | MT-Bench, AlpacaEval |
| 长文本 | 长序列处理 | LongBench, RULER |
| 安全性 | 安全对齐 | TruthfulQA, SafetyBench |

### Q: 评估方法分类

1. **自动评估**：固定 benchmark + 自动打分
2. **模型评估**：GPT-4/Claude 作为 judge
3. **人工评估**：人工打分/排序
4. **竞技场**：Chatbot Arena (ELO 排名)

### Q: LLM-as-Judge 的问题

- 位置偏差：倾向于选择第一个/第二个回答
- 自我偏好：倾向于自己模型的风格
- 长度偏差：倾向于更长的回答
- 解决方案：交换位置、多评估取平均
