# RAG 检索增强生成完全指南

## 1. RAG 基础

### Q: 什么是 RAG？⭐⭐⭐⭐

**RAG (Retrieval-Augmented Generation)** 通过外部知识检索增强 LLM 的生成能力。

**基本流程**：
```
用户查询 → 查询理解 → 检索相关文档 → 重排序 → 拼接 Prompt → LLM 生成
```

**RAG vs 微调**：

| 特性 | RAG | 微调 |
|------|-----|------|
| 知识更新 | 实时（更新索引） | 需要重新训练 |
| 幻觉控制 | 较好（有源可查） | 可能幻觉 |
| 成本 | 低 | 高 |
| 定制程度 | 中 | 高 |
| 推荐场景 | 知识密集型 | 风格/能力适配 |

## 2. 索引构建

### Q: 文档处理流程

```
原始文档 → 解析(PDF/HTML/...) → 清洗 → 分块(Chunking) → 向量化 → 存入向量数据库
```

### Q: Chunking 策略

| 策略 | 描述 | 适用场景 |
|------|------|----------|
| 固定大小 | 按 token 数切分 | 简单文本 |
| 句子级 | 按句子切分 | 精确检索 |
| 段落级 | 按段落切分 | 保持上下文 |
| 递归分割 | 按分隔符递归切分 | 通用（推荐） |
| 语义分割 | 按语义相似度切分 | 高质量 |

**Chunk 大小建议**：
- 通常 256-1024 tokens
- 添加 10-20% 的 overlap
- 根据下游任务调整

### Q: 常用 Embedding 模型

| 模型 | 维度 | 特点 |
|------|------|------|
| OpenAI text-embedding-3-large | 3072 | 商业最强 |
| BGE-M3 | 1024 | 多语言、多粒度 |
| E5-Mistral-7B-Instruct | 4096 | 指令增强 |
| GTE-Qwen2 | 1024 | 中英双语优秀 |

## 3. 检索策略

### Q: 向量检索 vs 关键词检索 vs 混合检索

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 向量检索 | 语义相似度 | 理解语义 | 精确匹配差 |
| 关键词(BM25) | 词频统计 | 精确匹配好 | 不理解语义 |
| **混合检索** | **两者结合** | **互补** | **需要融合** |

**混合检索融合**：
- RRF (Reciprocal Rank Fusion)：

$$\text{RRF}(d) = \sum_{r \in R} \frac{1}{k + r(d)}$$

### Q: 查询改写 (Query Rewriting)

1. **HyDE**：让 LLM 先生成一个假设性回答，用回答去检索
2. **Multi-Query**：将原始查询改写成多个不同表述
3. **Step-Back**：生成更通用的查询
4. **Sub-Query**：将复杂查询分解为子查询

## 4. 重排序 (Reranking)

### Q: 为什么需要重排序？

向量检索是 **bi-encoder**（双塔模型），速度快但精度有限。
重排序用 **cross-encoder**，query 和 doc 一起编码，精度更高。

```
检索 Top-K(快) → 重排序 Top-N(精) → 送入 LLM
```

常用重排序模型：
- Cohere Reranker
- BGE-Reranker-v2
- Cross-encoder/ms-marco

## 5. 高级 RAG 技术

### Q: Agentic RAG

让 Agent 决定何时检索、检索什么、如何使用检索结果：

```
用户查询
  → Agent 判断：需要检索吗？
    → 是：制定检索策略 → 检索 → 判断结果是否充分
      → 不充分：改写查询重新检索
      → 充分：生成回答
    → 否：直接回答
```

### Q: Graph RAG

利用知识图谱增强检索：
- 实体识别 → 构建知识图谱 → 图检索 + 向量检索
- 适合多跳推理和关系型查询

### Q: RAG 评估指标

| 维度 | 指标 | 衡量内容 |
|------|------|----------|
| 检索质量 | Recall@K, MRR, NDCG | 检索准确性 |
| 生成忠实度 | Faithfulness | 是否基于检索内容 |
| 答案相关性 | Answer Relevancy | 回答是否切题 |
| 上下文精确度 | Context Precision | 检索结果相关性 |

常用评估框架：RAGAS、TruLens
