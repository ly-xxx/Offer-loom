# 模型量化与部署

## 1. 量化基础

### Q: 什么是模型量化？

将模型权重和/或激活值从高精度（FP32/FP16）转换为低精度（INT8/INT4）。

### Q: PTQ vs QAT

| 方法 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| PTQ (训练后量化) | 训练完成后直接量化 | 简单快速 | 低比特精度损失大 |
| QAT (量化感知训练) | 训练时模拟量化 | 精度保持好 | 训练成本高 |

---

## 2. 常用量化方法

### GPTQ
- **原理**：基于 Hessian 矩阵的逐列量化
- **特点**：Weight-only 量化，3-4 bit，精度好
- **速度**：量化一次，推理用整数运算

### AWQ (Activation-aware Weight Quantization)
- **原理**：保护"重要"权重通道（根据激活值判断）
- **特点**：不需要反向传播，只需少量校准数据
- **效果**：4-bit 下比 GPTQ 更好

### GGUF (llama.cpp)
- **格式**：专为 CPU 推理设计的量化格式
- **支持**：Q2_K ~ Q8_0 多种量化级别
- **使用**：Ollama/llama.cpp 推理

---

## 3. 量化对比

| 方法 | 位宽 | 类型 | 适用场景 |
|------|------|------|----------|
| FP16 | 16-bit | 基线 | GPU 充足 |
| INT8 | 8-bit | PTQ | 轻度压缩 |
| GPTQ | 3/4-bit | PTQ | GPU 推理 |
| AWQ | 4-bit | PTQ | GPU 推理 |
| GGUF | 2-8 bit | PTQ | CPU 推理 |
| QLoRA | 4-bit | QAT | 微调 |

## 4. 部署架构

### 单机部署
```
客户端 → API Gateway → vLLM/TRT-LLM → GPU
```

### 分布式部署
```
客户端 → Load Balancer → [vLLM Instance 1 (TP=4)]
                       → [vLLM Instance 2 (TP=4)]
                       → [vLLM Instance 3 (TP=4)]
```

### 张量并行 vs 流水线并行

| 方式 | 原理 | 通信 | 适用 |
|------|------|------|------|
| 张量并行 (TP) | 单层切分到多卡 | AllReduce（频繁） | 多卡单机 |
| 流水线并行 (PP) | 不同层在不同卡 | 点对点（较少） | 跨机 |
